  <action>
Create the three core detection components in the WindowsHotSpot.Core namespace.

**HookManager.cs** (IDisposable):
- Class-level field `private readonly NativeMethods.LowLevelMouseProc _hookCallback;` initialized in constructor to prevent GC (CRITICAL -- Pitfall 2).
- `private IntPtr _hookId = IntPtr.Zero;`
- `public event Action&lt;Point&gt;? MouseMoved;` -- fires on WM_MOUSEMOVE with MSLLHOOKSTRUCT.pt converted to System.Drawing.Point.
- `public event Action&lt;bool&gt;? MouseButtonChanged;` -- fires true on WM_LBUTTONDOWN/WM_RBUTTONDOWN, false on WM_LBUTTONUP/WM_RBUTTONUP (for drag suppression, CORE-04).
- `public void Install()` -- gets module handle via Process.GetCurrentProcess().MainModule, calls NativeMethods.SetWindowsHookEx(WH_MOUSE_LL, ...). Throws Win32Exception on failure.
- `private IntPtr HookCallback(int nCode, IntPtr wParam, IntPtr lParam)` -- MUST be trivially fast (CRITICAL -- Pitfall 1). Only reads MSLLHOOKSTRUCT via Marshal.PtrToStructure, fires appropriate event, calls CallNextHookEx. No allocation, no I/O, no logging.
- `public void Dispose()` -- calls UnhookWindowsHookEx if _hookId != IntPtr.Zero, sets to IntPtr.Zero.
- Follow the exact pattern from 01-RESEARCH.md Pattern 2.

**ActionTrigger.cs** (static class):
- `public static void SendTaskView()` -- creates 4 INPUT structs (Win down, Tab down, Tab up, Win up) and calls SendInput atomically.
- Private helper `MakeKeyInput(ushort vk, bool keyUp)` returns an INPUT struct with type=INPUT_KEYBOARD, wVk set, dwFlags=KEYEVENTF_KEYUP if keyUp.
- Follow the exact pattern from 01-RESEARCH.md Pattern 5.

**CornerDetector.cs** (IDisposable):
- Enum `HotCorner { TopLeft, TopRight, BottomLeft, BottomRight }` -- define inside this file or as a nested type.
- Enum `DetectorState { Idle, Dwelling, Triggered }` -- three states (Idle, Dwelling, Triggered). Research shows 4 states but Cooldown is unnecessary -- Triggered already means "don't re-fire until mouse leaves zone", which transitions directly back to Idle.
- Private fields: `_state`, `_dwellTimer` (System.Windows.Forms.Timer -- NOT System.Timers.Timer, Pitfall from research), `_isButtonDown` (bool for drag suppression), `_activeCorner` (hard-coded to TopLeft for Phase 1), `_zoneSize` (hard-coded to 10 for Phase 1), `_dwellDelay` (hard-coded to 300 for Phase 1).
- Constructor: Create the timer, wire its Tick event to OnDwellComplete.
- `public void OnMouseMoved(Point pt)` -- called from HookManager.MouseMoved event. Implementation:
  1. Check if pt is in any screen's corner zone using Screen.AllScreens (CORE-02). For each screen, calculate the corner point using the GetCornerPoint pattern from 01-RESEARCH.md Pattern 4, then check if Math.Abs(pt.X - corner.X) &lt;= _zoneSize AND Math.Abs(pt.Y - corner.Y) &lt;= _zoneSize.
  2. State machine transitions per 01-RESEARCH.md Pattern 3:
     - Idle + in zone + !_isButtonDown -> start timer, state = Dwelling
     - Dwelling + NOT in zone -> stop timer, state = Idle
     - Dwelling + _isButtonDown -> stop timer, state = Idle (CORE-04)
     - Triggered + NOT in zone -> state = Idle (re-arm, CORE-05)
     - Triggered + in zone -> no-op (prevents double-trigger, CORE-05)
  3. Do NOT call ActionTrigger directly from this method (it is called from the hook callback chain and must be fast).
- `public void OnMouseButtonChanged(bool isDown)` -- sets _isButtonDown. If isDown and state == Dwelling, stop timer, state = Idle (CORE-04).
- `private void OnDwellComplete(object? sender, EventArgs e)` -- called when timer fires. Stop timer. Call ActionTrigger.SendTaskView(). Set state = Triggered. This runs on the UI thread (System.Windows.Forms.Timer) so SendInput is safe.
- `private static Point GetCornerPoint(Rectangle bounds, HotCorner corner)` -- switch expression per 01-RESEARCH.md Pattern 4.
- `public void Dispose()` -- dispose the timer.

IMPORTANT: The timer Tick handler (OnDwellComplete) is where ActionTrigger.SendTaskView() is called -- NOT in OnMouseMoved. OnMouseMoved only starts/stops the timer. The timer fires on the UI thread after the hook callback has already returned, keeping the callback fast.
