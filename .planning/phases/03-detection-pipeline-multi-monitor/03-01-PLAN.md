Add `MonitorCornerConfig` as an internal sealed class and `MonitorConfigs` as a new property on `AppSettings`.

In `AppSettings.cs`, after the `AppSettings` class closing brace, add:

```csharp
internal sealed class MonitorCornerConfig
{
    // All corners default to Disabled — new unrecognised monitors fire no actions (MMON-03).
    // Reuses DefaultCornerActions() helper to stay consistent with AppSettings defaults.
    public Dictionary<HotCorner, CornerAction> CornerActions { get; set; }
        = AppSettings.DefaultCornerActions();
}
```

On `AppSettings`, add the new property after `DwellDelayMs`:

```csharp
// Per-monitor corner config keyed by Screen.DeviceName.
// Missing key = use global CornerActions fallback.
// Entries for disconnected monitors are retained indefinitely (MMON-04).
public Dictionary<string, MonitorCornerConfig> MonitorConfigs { get; set; } = new();
```

Do NOT change CornerActions, DefaultCornerActions(), LegacyCorner, or any existing properties.

Why string key (not HotCorner key): System.Text.Json serializes string-keyed dictionaries without a custom converter. Enum-keyed dictionaries require extra converter configuration. Screen.DeviceName (e.g., `"\\.\DISPLAY1"`) is the stable GDI device string — more stable than monitor index which can change when display order is reordered.

Why no [JsonConverter] on MonitorCornerConfig: The nested CornerActions dictionary inside MonitorCornerConfig uses HotCorner enum keys. This already works because the existing [JsonConverter(typeof(JsonStringEnumConverter))] on the HotCorner enum declaration causes enum keys to serialize as strings in dictionaries when using the JsonSerializerOptions with JsonStringEnumConverter already configured in ConfigManager. Verify ConfigManager's JsonSerializerOptions includes this behavior before assuming.

Rewrite CornerDetector to accept `Rectangle screenBounds` and `CornerAction action` at construction instead of `ConfigManager configManager`. This eliminates shared mutable state between detector generations and scopes the zone check to one screen only.

Constructor change:
```csharp
// Before (Phase 2):
public CornerDetector(HotCorner corner, int zoneSize, int dwellDelay, ConfigManager configManager)

// After (Phase 3):
public CornerDetector(HotCorner corner, Rectangle screenBounds, int zoneSize, int dwellDelay, CornerAction action)
```

Fields to change:
- Remove: `private readonly ConfigManager _configManager;`
- Add: `private readonly Rectangle _screenBounds;`
- Add: `private readonly CornerAction _action;`
- Keep: `_activeCorner`, `_zoneSize`, `_dwellDelay`, `_state`, `_isButtonDown`, `_dwellTimer` (unchanged)

Remove the `UpdateSettings` method entirely. CornerRouter rebuilds the detector pool on any change — in-place mutation is no longer the pattern.

Rename `IsInAnyCornerZone` to `IsInCornerZone` and change implementation:
```csharp
// Before: iterates all Screen.AllScreens
private bool IsInAnyCornerZone(Point pt)
{
    foreach (var screen in Screen.AllScreens) { ... }
}

// After: checks only this detector's screen
private bool IsInCornerZone(Point pt)
{
    Point corner = GetCornerPoint(_screenBounds, _activeCorner);
    return Math.Abs(pt.X - corner.X) <= _zoneSize
        && Math.Abs(pt.Y - corner.Y) <= _zoneSize;
}
```

Update `OnMouseMoved` to call `IsInCornerZone` (not `IsInAnyCornerZone`).

Update `OnDwellComplete` to use `_action` directly:
```csharp
private void OnDwellComplete(object? sender, EventArgs e)
{
    _dwellTimer.Stop();
    ActionDispatcher.Dispatch(_action);   // _action fixed at construction; no ConfigManager lookup
    _state = DetectorState.Triggered;
}
```

Remove the `using WindowsHotSpot.Config;` import only if ConfigManager is the only thing it was needed for — HotCorner, CornerAction, and CornerDetector all live in the Config namespace, so keep the using.

Update the file header comment to: "Phase 3: Constructor now takes Rectangle screenBounds + CornerAction; ConfigManager dependency removed. Zone check scoped to _screenBounds only (not Screen.AllScreens). UpdateSettings() removed — CornerRouter rebuilds the detector pool on changes."

Do NOT change: OnMouseButtonChanged, GetCornerPoint, Dispose, DetectorState enum.
