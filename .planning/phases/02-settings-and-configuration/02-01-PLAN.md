  <action>
Create the `WindowsHotSpot/Config/` directory with three files:

**AppSettings.cs:**
- Internal sealed class `AppSettings` in namespace `WindowsHotSpot.Config`
- Properties: `Corner` (HotCorner, default TopLeft), `ZoneSize` (int, default 10), `DwellDelayMs` (int, default 300), `StartWithWindows` (bool, default false)
- Move the `HotCorner` enum from `Core/CornerDetector.cs` to this file (same namespace `WindowsHotSpot.Config`)
- Add `[JsonConverter(typeof(JsonStringEnumConverter))]` attribute to HotCorner enum so JSON serializes as "TopLeft" not "0"

**ConfigManager.cs:**
- Internal sealed class `ConfigManager` in namespace `WindowsHotSpot.Config`
- Static readonly `SettingsPath` = `Path.Combine(AppContext.BaseDirectory, "settings.json")`
- Static readonly `JsonOptions` with `WriteIndented = true` and `JsonStringEnumConverter` in Converters
- Property `Settings` (AppSettings, getter public, setter private, initialized to `new()`)
- Event `Action? SettingsChanged`
- Method `Load()`: if settings.json exists, read and deserialize; wrap in try/catch, fall back to `new AppSettings()` on any exception (corrupt file resilience per research Pitfall 2). Also: if `Settings.StartWithWindows` is true, call `StartupManager.RefreshPath()` to fix stale registry path after exe move (research Pitfall 3)
- Method `Save()`: serialize Settings to JSON, write to SettingsPath, invoke SettingsChanged event

**StartupManager.cs:**
- Internal static class `StartupManager` in namespace `WindowsHotSpot.Config`
- Constants: `RunKey = @"Software\Microsoft\Windows\CurrentVersion\Run"`, `AppName = "WindowsHotSpot"`
- Static property `IsEnabled` (get): open HKCU RunKey readonly, return whether AppName value exists
- Static method `SetEnabled(bool enable)`: open HKCU RunKey writable; if enable, set value to quoted `Environment.ProcessPath`; if disable, DeleteValue with throwOnMissingValue: false
- Static method `RefreshPath()`: if IsEnabled, call SetEnabled(true) to update the path (handles exe move per research Pitfall 3)
- Use `Microsoft.Win32.Registry` (inbox, NOT P/Invoke)
- Use `Environment.ProcessPath` (NOT `Assembly.Location` which returns empty in single-file publish)
