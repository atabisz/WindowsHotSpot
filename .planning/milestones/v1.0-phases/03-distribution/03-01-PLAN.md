<task type="auto">
  <name>Task 1: Add publish properties to csproj and create app icon</name>
  <files>
    WindowsHotSpot/WindowsHotSpot.csproj
    WindowsHotSpot/Resources/app.ico
  </files>
  <action>
1. Create `WindowsHotSpot/Resources/` directory.

2. Generate a minimal app.ico file. Use PowerShell and System.Drawing to create a simple 256x256 icon programmatically:
   - Create a 256x256 Bitmap with a colored background (use a blue gradient or solid color like #0078D4 -- the Windows accent blue)
   - Draw a simple corner indicator shape: a small white right-angle bracket in the top-left corner area (representing a "hot corner")
   - Save as ICO format using System.Drawing.Icon. The ICO must contain at least 16x16, 32x32, 48x48, and 256x256 sizes.
   - Alternative if programmatic ICO creation is too complex: use `magick` or download a simple placeholder. As a last resort, use the .NET favicon trick: create a 256x256 PNG and convert to ICO using a PowerShell script that writes the ICO binary header manually (a single-image ICO is: 6-byte header + 16-byte directory entry + PNG data).
   - The key requirement is that app.ico exists and is a valid ICO file.

3. Modify `WindowsHotSpot/WindowsHotSpot.csproj` -- add these properties to the existing PropertyGroup (do NOT create a second PropertyGroup, add to the existing one):
   ```xml
   <RuntimeIdentifier>win-x64</RuntimeIdentifier>
   <PublishSingleFile>true</PublishSingleFile>
   <SelfContained>true</SelfContained>
   <EnableCompressionInSingleFile>true</EnableCompressionInSingleFile>
   <IncludeNativeLibrariesForSelfExtract>true</IncludeNativeLibrariesForSelfExtract>
   <ApplicationIcon>Resources\app.ico</ApplicationIcon>
   ```

   Do NOT add PublishTrimmed (risky for WinForms per research). Do NOT remove any existing properties. The existing PropertyGroup already has OutputType, TargetFramework, Nullable, UseWindowsForms, ImplicitUsings, ApplicationHighDpiMode, ApplicationManifest, and NoWarn.

4. Verify `dotnet build WindowsHotSpot/WindowsHotSpot.csproj -c Release` succeeds (proves the icon and csproj changes are valid without doing a full publish yet).
  </action>
  <verify>
    <automated>cd C:/src/WindowsHotSpot &amp;&amp; dotnet build WindowsHotSpot/WindowsHotSpot.csproj -c Release 2>&amp;1 | tail -5</automated>
    <manual>Check that WindowsHotSpot/Resources/app.ico exists and is > 0 bytes</manual>
  </verify>
  <done>csproj contains all 6 publish properties (RuntimeIdentifier, PublishSingleFile, SelfContained, EnableCompressionInSingleFile, IncludeNativeLibrariesForSelfExtract, ApplicationIcon). app.ico exists in Resources/ and is a valid ICO file. Release build succeeds.</done>
</task>

<task type="auto">
  <name>Task 2: Create build script and verify self-contained publish</name>
  <files>
    build.ps1
  </files>
  <action>
1. Create `build.ps1` in the repo root with the following content:
   - Set `$ErrorActionPreference = "Stop"`
   - Define `$publishDir = "WindowsHotSpot\bin\Release\net10.0-windows\win-x64\publish"`
   - Run `dotnet publish WindowsHotSpot\WindowsHotSpot.csproj -c Release`
   - Check `$LASTEXITCODE`, throw on failure
   - Print the published exe path and size in MB (using `Get-Item` and `.Length / 1MB`)
   - Check for ISCC.exe at `"C:\Program Files (x86)\Inno Setup 6\ISCC.exe"`:
     - If found: run `& $iscc "installer\WindowsHotSpot.iss"` and check exit code
     - If not found: print yellow warning "Inno Setup not found. Install with: choco install innosetup -y" and "Skipping installer build."
   - Use the exact script from research Pattern 3, adapted as needed.

2. Run `dotnet publish WindowsHotSpot/WindowsHotSpot.csproj -c Release` to verify it produces a single exe.

3. Verify the publish output:
   - Exactly ONE .exe file should exist in the publish directory
   - No loose .dll files should be present (they should all be bundled)
   - The exe should be > 10 MB (self-contained WinForms apps are typically 30-60 MB with compression)

4. Run the published exe briefly to confirm it starts (it should create a tray icon). Kill it after confirming it launches without errors.
