## Overview

v1.6 adds a fourth Ctrl+Alt gesture: hold **Ctrl+Alt+Shift** and scroll the mouse wheel over any window to adjust its transparency. Scroll down reduces opacity; scroll up increases it. Alpha is clamped to 25–255 so windows can never become invisible or fully disappear.

The chord is deliberately distinct from scroll resize (Ctrl+Alt, no Shift) — `ScrollResizeHandler` was updated to exclude LShift from its activation gate so the two gestures cannot fire simultaneously.

The transparency step size is user-configurable in **Settings → Window Interactions** (default 10 α/notch, range 1–50).

**Details:**

- `VK_LSHIFT` case has no `!isInjected` guard — AltGr does not synthesize LShift (unlike LCtrl)
- 5-step alpha pipeline: read `GWL_EXSTYLE` → `GetLayeredWindowAttributes` → compute new alpha → set `WS_EX_LAYERED` if absent → `SetLayeredWindowAttributes` with `existingFlags | LWA_ALPHA`
- `LWA_COLORKEY` preserved: OR-with-existingFlags ensures color-key windows are not broken
- Alpha clamped: `Math.Clamp(existingAlpha + step, 25, 255)` — 25 minimum prevents invisible windows

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Clone `ScrollResizeHandler` exactly | Proven pattern; consistent keyboard hook ownership model | ✓ Good |
| No `!isInjected` guard on VK_LSHIFT | AltGr synthesizes fake LCtrl but not LShift; guard would be dead code | ✓ Good |
| OR-lambda for `WheelSuppressionPredicate` | Field is single-slot (not event); OR-lambda is the only way to combine two handlers | ✓ Good |
| `LWA_COLORKEY` preserved via `existingFlags \| LWA_ALPHA` | Read-then-OR ensures color-key windows keep their flags after transparency change | ✓ Good |
| Alpha min = 25 (not 0 or 1) | Prevents windows from becoming invisible and unrecoverable | ✓ Good |
| `IsPhysicallyDown(VK_LSHIFT)` in `ScrollResizeHandler` gate | No Shift state tracking needed — live query at wheel-fire time is sufficient | ✓ Good |
| TransparencyStep range 1–50 in Settings | Matches practical alpha units (255 range / 50 = 5.1 min steps to full sweep) | ✓ Good |

**Issues Resolved:**
- Ctrl+Alt+Shift+scroll was simultaneously resizing and adjusting transparency — fixed in `ScrollResizeHandler` post-ship

**Issues Deferred:**
- Per-window transparency memory (restore alpha on close/reopen) — different product surface
- Visual indicator of current transparency level — needs overlay or tray tooltip
- `IsElevatedProcess` / `IsPhysicallyDown` still duplicated across four handlers — shared utility extraction deferred

**Technical Debt Incurred:**
- Fourth handler (`WindowTransparencyHandler`) repeats the `IsElevatedProcess` / `IsPhysicallyDown` helper pattern — now in drag, resize, AOT, and transparency handlers
