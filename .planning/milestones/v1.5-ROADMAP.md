## Overview

v1.5 extends the Ctrl+Alt window-interaction model established in v1.4 (drag-to-move) with two new gestures:
- **Scroll resize** — Ctrl+Alt+scroll wheel grows or shrinks the window under the cursor, symmetrically around the cursor position, with configurable step size (Settings → Window Interactions)
- **Always-on-top toggle** — Ctrl+Alt+double-click pins or unpins a window to the top of the Z-order, with a tray balloon confirming the state using the target window's title

Both gestures share the LCtrl+LAlt modifier guard (AltGr excluded via `LLKHF_INJECTED`), maximized-window skip, and UIPI elevated-window skip — consistent with drag behaviour.

A hook infrastructure phase (Phase 8) extended `HookManager` with `MouseWheeled` and `MouseDoubleClicked` events, and added `WheelSuppressionPredicate` to prevent console font zoom during scroll resize.

## Phases

### Phase 8: Hook Infrastructure

**Goal**: HookManager delivers wheel and double-click events to consumers
**Depends on**: Phase 7 (wiring)
**Plans**: 1 plan

Plans:
- [x] 08-01: Add `MouseWheeled (Action<int,Point>?)` and `MouseDoubleClicked (Action<Point>?)` events; WM_MOUSEWHEEL branch with signed HIWORD delta; double-click detection via uint-rollover-safe time+distance tracking against `GetDoubleClickTime`/`SM_CXDOUBLECLK`/`SM_CYDOUBLECLK`

**Details:**
- NativeMethods additions: `WM_MOUSEWHEEL (0x020A)`, `WM_LBUTTONDBLCLK (0x0203)`, `SM_CXDOUBLECLK (36)`, `SM_CYDOUBLECLK (37)`, `GetDoubleClickTime()`, `GetSystemMetrics(int)`
- Double-click detection owned by HookManager (not consumers) — single implementation, consistent behaviour
- Tracking state resets after detection so triple-click does not re-fire
- `SuppressionPredicate` early return gates the detection block — suppressed clicks never feed tracking state

Post-ship addition: `WheelSuppressionPredicate` added to prevent WM_MOUSEWHEEL passthrough to console apps (which zoom font on Ctrl+scroll)

### Phase 9: Scroll Resize

**Goal**: Users can resize any eligible window by holding Ctrl+Alt and scrolling
**Depends on**: Phase 8
**Plans**: 2 plans

Plans:
- [x] 09-01: `ScrollResizeHandler` class — WH_KEYBOARD_LL hook, `OnMouseWheeled`, cursor-anchored resize math, SM_CXMINTRACK/SM_CYMINTRACK clamp; `AppSettings.ScrollResizeStep = 20`
- [x] 09-02: SettingsForm "Window Interactions" GroupBox (NumericUpDown 1–200 px/notch); `HotSpotApplicationContext` wiring

**Details:**
- Cursor-anchored math: `fx = (cursorX - left) / w`, `newLeft = cursorX - (fx * nw)` — cursor's fractional window position preserved after resize
- Step: `delta * ScrollResizeStep / 120` — scales correctly for high-resolution scroll wheels
- Guards: skip maximized (`SW_SHOWMAXIMIZED`), skip elevated (`IsElevatedProcess`), zero-size window (`w==0||h==0`)
- Screen clamping: each edge independently clamped to `Screen.WorkingArea` (taskbar excluded); minimum size re-applied after clamping

Post-ship fixes:
1. Suppress WM_MOUSEWHEEL passthrough via `WheelSuppressionPredicate` — prevents console font zoom
2. Per-edge screen clamping — stop growing at monitor boundary
3. Use `WorkingArea` not `Bounds` — respect taskbar boundary

### Phase 10: Always-on-Top Toggle

**Goal**: Users can pin any eligible window to the top of the Z-order with a double-click gesture
**Depends on**: Phase 8
**Plans**: 2 plans

Plans:
- [x] 10-01: `AlwaysOnTopHandler` class — WH_KEYBOARD_LL hook, self-tracked double-click detection via `MouseButtonChanged` (fires before suppression), `GetWindowLong(GWL_EXSTYLE) & WS_EX_TOPMOST` to read state, `SetWindowPos(HWND_TOPMOST/HWND_NOTOPMOST)` to toggle, `ShowBalloonTip` with target window title; NativeMethods additions: `SWP_NOMOVE`, `HWND_TOPMOST`, `HWND_NOTOPMOST`, `GWL_EXSTYLE`, `WS_EX_TOPMOST`, `GetWindowLong`, `GetWindowText`
- [x] 10-02: `HotSpotApplicationContext` wiring — field, constructor (instantiate + subscribe `MouseButtonChanged` + `Install`), `DisposeComponents` (unsubscribe-before-dispose)

**Details:**
- AOT uses `MouseButtonChanged` (not `MouseDoubleClicked`) because Ctrl+Alt clicks are suppressed by drag, so the double-click block in HookManager never fires for Ctrl+Alt clicks. Handler self-tracks timing/position.
- `_lastDownTime != 0` guard: first click never falsely triggers
- Tray balloon title uses target window's `GetWindowText` result (not "WindowsHotSpot") — "Notepad: Pinned on top"

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| `WheelSuppressionPredicate` separate from `SuppressionPredicate` | Wheel suppression is consumer-gated (only when Ctrl+Alt held), not always-on like button suppression | ✓ Good |
| Double-click detection in `AlwaysOnTopHandler` via `MouseButtonChanged`, not `MouseDoubleClicked` | `MouseDoubleClicked` fires after suppression — Ctrl+Alt clicks are suppressed, so it never fires | ✓ Good |
| `Screen.WorkingArea` for scroll resize clamping | `Bounds` includes taskbar; `WorkingArea` excludes it — window stops at taskbar boundary | ✓ Good |
| Per-edge clamping (independent left/right/top/bottom) | Whole-window clamp would refuse resize when any edge is at boundary; per-edge allows partial grow | ✓ Good |
| `GetWindowText` for AOT balloon title | Shows the target app's name, making feedback immediately meaningful ("Notepad" vs "WindowsHotSpot") | ✓ Good |
| Three separate WH_KEYBOARD_LL hooks (drag, resize, AOT) | Each handler owns its modifier state independently; no shared state coupling | ✓ Good |
| `SWP_ASYNCWINDOWPOS` without `SWP_NOSIZE` in resize | Resize must pass `cx`/`cy`; async prevents blocking hook if target is slow | ✓ Good |

**Issues Resolved:**
- Console font zoom during Ctrl+Alt scroll — `WheelSuppressionPredicate` suppresses WM_MOUSEWHEEL when Ctrl+Alt held
- Window growing below taskbar — `Screen.WorkingArea` replaces `Screen.Bounds`
- Window growing off screen edges — per-edge clamping added

**Issues Deferred:**
- Per-app exclusion list for resize/AOT — different product surface
- Visual indicator on pinned windows (title bar tint) — requires accessibility tree work
- Resize step configurable per-axis — over-engineering for v1.5

**Technical Debt Incurred:**
- `IsElevatedProcess` and `IsPhysicallyDown` helpers are duplicated across `WindowDragHandler`, `ScrollResizeHandler`, and `AlwaysOnTopHandler` — candidate for extraction to a shared utility class in a future refactor phase
