# Requirements Archive: WindowsHotSpot v1.5 Window Interactions

**Archived:** 2026-05-06
**Status:** ‚úÖ All requirements shipped

## Milestone v1.5 Requirements

### Hook Infrastructure

- [x] **HOOK-03**: HookManager handles WM_MOUSEWHEEL events and fires a new `MouseWheeled` event with delta and cursor position
  - *Outcome: Delivered in Phase 8 (08-01). `MouseWheeled (Action<int,Point>?)` fires on every WM_MOUSEWHEEL; delta extracted as `(short)(mouseData >> 16)` for correct signed polarity.*
- [x] **HOOK-04**: HookManager handles WM_LBUTTONDBLCLK events and fires via the existing `MouseButtonChanged` event or a new dedicated event
  - *Outcome: Delivered in Phase 8 (08-01). `MouseDoubleClicked (Action<Point>?)` fires on two consecutive WM_LBUTTONDOWN within system double-click time+distance thresholds. Note: AlwaysOnTopHandler (Phase 10) self-tracks double-click via `MouseButtonChanged` because Ctrl+Alt clicks are suppressed before the detection block runs.*

### Scroll Resize

- [x] **RESIZE-01**: User can hold Ctrl+Alt and scroll the mouse wheel to resize the window under the cursor ‚Äî scroll up grows, scroll down shrinks
  - *Outcome: Delivered in Phase 9. `ScrollResizeHandler.OnMouseWheeled` gates on `_lCtrlDown && _lAltDown`; step sign from delta polarity.*
- [x] **RESIZE-02**: Resize is applied symmetrically to all four edges, anchored around the cursor position
  - *Outcome: Delivered in Phase 9. Cursor fractional position `fx/fy` preserved; `newLeft = cursorX - (fx * nw)`, `newTop = cursorY - (fy * nh)`.*
- [x] **RESIZE-03**: Step size per scroll notch is configurable in Settings (default 20px); the setting persists across restarts
  - *Outcome: Delivered in Phase 9 (09-01 + 09-02). `AppSettings.ScrollResizeStep = 20`; SettingsForm "Window Interactions" GroupBox with NumericUpDown (1‚Äì200 px/notch); persists via `ConfigManager.Save()`.*
- [x] **RESIZE-04**: Resize is silently clamped at Windows minimum window size ‚Äî no error, no feedback
  - *Outcome: Delivered in Phase 9. `Math.Max(nw, minW)` / `Math.Max(nh, minH)` using `SM_CXMINTRACK (34)` / `SM_CYMINTRACK (35)`. Re-applied after screen clamping.*
- [x] **RESIZE-05**: Scroll resize is skipped for maximized windows
  - *Outcome: Delivered in Phase 9. `GetWindowPlacement` check; skip if `showCmd == SW_SHOWMAXIMIZED`.*
- [x] **RESIZE-06**: Scroll resize is skipped for elevated (admin) windows ‚Äî UIPI constraint
  - *Outcome: Delivered in Phase 9. `IsElevatedProcess` guard (identical to `WindowDragHandler`).*
- [x] **RESIZE-07**: AltGr (RCtrl+LAlt) does not trigger scroll resize ‚Äî only LCtrl+LAlt activates it
  - *Outcome: Delivered in Phase 9. `LLKHF_INJECTED` guard on `VK_LCONTROL` in `KeyboardCallback` ‚Äî same as GUARD-01 in drag.*

### Always-on-Top Toggle

- [x] **AOT-01**: User can Ctrl+Alt+double-click a window to toggle its always-on-top state
  - *Outcome: Delivered in Phase 10. `AlwaysOnTopHandler` self-tracks double-click; `SetWindowPos(HWND_TOPMOST/HWND_NOTOPMOST)` toggles Z-order.*
- [x] **AOT-02**: A tray balloon tooltip briefly appears confirming the state change
  - *Outcome: Delivered in Phase 10. `ShowBalloonTip(2000, windowTitle, "Pinned on top"/"Unpinned", Info)` where `windowTitle` is from `GetWindowText`.*
- [x] **AOT-03**: Always-on-top toggle is skipped for elevated (admin) windows ‚Äî UIPI constraint
  - *Outcome: Delivered in Phase 10. `IsElevatedProcess` guard in `ToggleAlwaysOnTop`.*
- [x] **AOT-04**: AltGr (RCtrl+LAlt) does not trigger the toggle ‚Äî only LCtrl+LAlt activates it
  - *Outcome: Delivered in Phase 10. `LLKHF_INJECTED` guard on `VK_LCONTROL` in `KeyboardCallback`.*

## Requirements Adjusted During Milestone

- **AOT-02 balloon title**: Original spec said "WindowsHotSpot: Pinned on top". Adjusted during testing to use the target window's title (via `GetWindowText`) for more useful feedback. No requirement conflict ‚Äî improvement to the original intent.
- **HOOK-04 (MouseDoubleClicked) not used by AOT**: The event was added as specified, but `AlwaysOnTopHandler` uses `MouseButtonChanged` instead due to the suppression-pipeline constraint. The event remains available for other consumers.
