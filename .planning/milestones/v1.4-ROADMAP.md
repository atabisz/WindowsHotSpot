## Overview

Ctrl+Alt+drag to move any restored window without grabbing the title bar. Includes suppression infrastructure in HookManager, complete WindowDragHandler implementation, AltGr protection, maximized-window skip, elevated-window detection, and SAS self-heal to prevent lockup after Ctrl+Alt+Del.

**Depends on**: Nothing (extends existing HookManager)

**Plans**: 1 plan

**Goal**: Users can hold Ctrl+Alt and drag any non-maximized window to move it, with smooth real-time tracking, clean release, and AltGr protection
**Depends on**: Phase 5

Plans:

- [x] 06-03-PLAN.md — Create WindowDragHandler.cs with complete drag implementation (Wave 2)
- [x] 06-04-PLAN.md — Temporary wiring smoke test and manual verification checkpoint (Wave 3, checkpoint approved 2026-05-05)

**Success Criteria (all met):**
1. Holding LCtrl+LAlt + left-mouse on any restored window begins drag — window moves with cursor delta in real time
2. Topmost window under cursor is selected (not necessarily the foreground window)
3. Initiating click is not delivered to the target application while drag mode is active
4. Releasing left mouse button ends the drag cleanly
5. Clicking on a maximized window while holding LCtrl+LAlt does not start a drag — click passes through
6. Holding RCtrl+LAlt (AltGr) does not trigger drag — only LCtrl+LAlt activates it
7. If Ctrl or Alt is released during active drag, drag is cancelled and window stays at current position

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| SuppressionPredicate as `Func<int, bool>?` property | Consumer registers inline; null short-circuits to false with `?.Invoke == true` — no branch overhead | ✓ Good |
| MouseButtonChanged fires BEFORE SuppressionPredicate | Consumer state must be current when predicate runs — event-before-predicate ordering eliminates race | ✓ Good |
| WH_KEYBOARD_LL in WindowDragHandler (not HookManager) | Keyboard hook is drag-specific; separating concerns avoids polluting HookManager with drag state | ✓ Good |
| Absolute-delta position math | `newX = _windowOrigin.Left + (pt.X - _dragStartCursor.X)` — no accumulated drift vs per-frame delta | ✓ Good |
| SWP_ASYNCWINDOWPOS in SetWindowPos | Prevents blocking hook callback if target app's message pump is slow | ✓ Good |
| LLKHF_INJECTED guard for VK_LCONTROL | AltGr sends a fake VK_LCONTROL before VK_RMENU — injected bit distinguishes it | ✓ Good |
| _suppressNextClick cleared on WM_LBUTTONUP (not WM_LBUTTONDOWN) | Full button pair (down+up) must be suppressed; clearing on down left orphaned LBUTTONUP reaching target | ✓ Good |
| Maximized window: return with no _suppressNextClick | DRAG-06 requires click pass-through on maximized windows — setting _suppressNextClick would break that | ✓ Good |
| GetKeyState self-heal on BeginDragAttempt | SAS (Ctrl+Alt+Del) swallows key-up events; tracked state can get stuck true — physical key state check self-heals | ✓ Good |
| IsElevatedProcess check before committing drag | UIPI blocks SetWindowPos cross-elevation; detecting upfront lets click pass through cleanly instead of consuming and failing silently | ✓ Good |
| WindowDragHandler as readonly non-nullable field | Field initialized in constructor — nullable + null-guard was unnecessary scaffolding from Phase 6 temp wiring | ✓ Good |

**Issues Resolved:**

- CR-01: `WindowDragPassThrough` setting was never saved — added `_configManager.Settings.WindowDragPassThrough = form.SelectedWindowDragPassThrough` in ShowSettingsWindow OK path
- CR-02: `_suppressNextClick` cleared on LBUTTONDOWN, leaving orphaned LBUTTONUP reaching target — fixed to clear on LBUTTONUP only
- WR-01: `NumericUpDown` value set before min/max established, throwing `ArgumentOutOfRangeException` — fixed with `Math.Clamp`
- WR-02: `GetWindowRect` failure path returned without setting `_suppressNextClick` — made consistent with all other failure paths
- WR-03: Double-dispose vulnerability in `DisposeComponents` — added `_disposed` guard
- WR-04: `SettingsChanged` lambda stored as anonymous delegate, preventing unsubscription — stored in field
- WR-05: `CancelDragIfActive` duplicated `EndDrag` logic — refactored to delegate
- Bug (post-ship): SAS lockup — modifier state stuck true after Ctrl+Alt+Del, locking all clicks — fixed with GetKeyState self-heal
- Bug (post-ship): Task Manager / elevated windows — UIPI silently blocks SetWindowPos; fixed with elevation detection
